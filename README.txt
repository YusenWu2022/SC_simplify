SC2_simplify 小介绍
                         by 吴宇森2000013137
1、平日里对星际争霸2比较有兴趣，并且发现即时战略游戏数目众多的单位和技能很适合使用课上学到的关于Java的类和对象编程方式实现；
同时飞行，埋地，能量，护甲类型等属性则适合用接口描述。这样，一个单位就可以用一个逐渐加深的抽象父类和不断加入的接口具象化；
最后再加上独有的特性，比如控制，闪电等就构造了单位对象。
2、完成了一部分单位类的定义，并在主函数里面新建了几个刺蛇和飞龙,使他们相互攻击，还测试了用单位数组作为参数的感染虫对群体
（数组）对象的集体扣血处理，这里使用了增强for循环。
3、所有的变量（域）都设为私有，并通过共有的查找和修改 函数进行间接但安全的访问（getter和setter)
使用了@override 标签表示对父类方法的覆盖
4、未来可能完善三族单位，进一步引入UI界面，设计玩家交互界面，并利用Java多线程复现原版游戏中的实时操作，配置服务器多线程，实现
简单的多人游戏逻辑和对星际游戏的simplify复现，应该还是一个比较大的项目吧。
5、具体介绍
项目包SC2_simplify 下面mainUnit包储存类内容，MainWorld文件运行主程序。mainUnit下属Armors包定义了护甲接口，material定
义了单位类型，在核心的Unit包中分别定义了抽象类Unit，ZergUnit,ProtossUnit等，以及对应的实体类Hutalisk,Hydralisk，zealot
等等。在调试方面，在Unit中定义了static的TestMode参数，只需改动这个值就可以控制调试语句是否输出。
补充：除了直接生成全局实例，还完成了player类的构造，可以在该类实例中完成单位的生成和控制，相当于一个控制器，进一步去耦合，
main函数下只需要维护map地图类实例和多个player实例来进行操作就可以了，减少了错误的情况。
